SCHEDULING ALGORITHMS

CPU Scheduling is a process of determining which process will own CPU for execution while another process is on hold. The main task of CPU scheduling is to make sure that whenever the CPU remains idle, the OS at least select one of the processes available in the ready queue for execution. The selection process will be carried out by the CPU scheduler. It selects one of the processes in memory that are ready for execution.

First Come First Serve

First come first serve (FCFS) scheduling algorithm simply schedules the jobs according to their arrival time. The job which comes first in the ready queue will get the CPU first. The lesser the arrival time of the job, the sooner will the job get the CPU. FCFS scheduling may cause the problem of starvation if the burst time of the first process is the longest among all the jobs.

Advantages Disadvantages
Simple & Easy Large Average Waiting Time
Starvation Doesnt Occurs Does't Work For Priority Process
Self Explanatory Logic Not Efficient

Non Preemptive Shortest Job First

Shortest job first (SJF) or shortest job next, is a scheduling policy that selects the waiting process with the smallest execution time to execute next. SJN is a non-preemptive algorithm. Shortest Job first has the advantage of having a minimum average waiting time among all scheduling algorithms.

Advantages Disadvantages
Optimal Turn Around Time Requires knowledge of how long a process or job will run
minimum average waiting time It is often used in a batch system for long term scheduling
Suitable For Long Term Scheduling Starvation May Happens

Non Preemptive Priority Scheduling

Priority Scheduling is a method of scheduling processes that is based on priority. In this algorithm, the scheduler selects the tasks to work as per the priority. The processes with Lower priority should be carried out first, whereas jobs with equal priorities are carried out on a round-robin or FCFS basis. Priority depends upon memory requirements, time requirements, etc.

Advantages Disadvantages
Once resources are allocated to a process, the process holds it till it completes its burst time even if a process with higher priority is added to the queue. Bugs can cause a machine to freeze up
It is cheaper to implement and faster as less switching is required. Bugs can cause a machine to freeze up
It can be used in various hardware applications where waiting will not cause any serious issues. It can lead to starvation especially for those real-time tasks.

Preemptive Shortest Job First

In Preemptive SJF Scheduling, jobs are put into the ready queue as they come. A process with shortest burst time begins execution. If a process with even a shorter burst time arrives, the current process is removed or preempted from execution, and the shorter job is allocated CPU cycle..

Advantages Disadvantages
Optimal Turn Around Time Requires knowledge of how long a process or job will run
minimum average waiting time It is often used in a batch system for long term scheduling
Suitable For Long Term Scheduling Starvation May Happens

Round Robin

Round Robin is a CPU scheduling algorithm where each process is assigned a fixed time slot in a cyclic way. It is simple, easy to implement, and starvation-free as all processes get fair share of CPU. One of the most commonly used technique in CPU scheduling as a core

Advantages Disadvantages
The relative importance of each process maybe precisely defined. problem is deciding which process gets which priority level assigned to it
Optimal Algorithms for lesser Tat And Wt Starvation May Cause By Some High Priority Process
Suitable For Long Term Scheduling & Short Scheduling. If slicing time of OS is low, the processor output will be reduced
